# Multi-Device Recording System

---

## 1. System Description

A multi-device recording system coordinated by a Pi 4 controller. All devices record synchronously under a shared session UUID.

**Devices**

| ID | Role |
|---|---|
| `ctlr` | Pi 4 — session authority, data aggregation |
| `cam-01` | Pi Zero 2 — 1080p video, 360p live stream to Pi 4 |
| `phone-01` | iPhone — sensor recording + user UI |
| `watch-01` | Apple Watch — sensor recording + user UI |
| `sensor-01` | Pi 4 onboard — CAN, GNSS, IMU |

**Network:** Tailscale VPN

---

## 2. Naming Conventions

**MQTT topic structure**
```
<site>/<controller>/<device-type>/<device-id>/<data-type>/<uuid>
```

**Site:** `melb-01`
**Device types:** `cam` · `phone` · `watch` · `can` · `imu` · `gnss`
**Device IDs:** `cam-01` · `phone-01` · `watch-01` · `can-01` · `imu-01` · `gnss-01`

**Session UUID:** UUIDv4, generated by controller at session start, shared across all devices and used as a key in all data storage.

---

## 3. Session Lifecycle

**States:** `idle → preflight → recording`

1. User taps **Start** — Pi 4 accepts only from `idle`
2. Pi 4 generates UUID, publishes preflight command; all devices must confirm ready
3. Pi 4 publishes `session/start` with UUID and `start_time` (+5s)
4. All devices begin recording at `start_time`
5. User taps **Stop** — all devices stop, Pi 4 triggers rsync + upload

**Crash / late join:** Retained topics `session/state` and `session/last` allow devices to rejoin an active session automatically on reconnect.

---

## 4. MQTT Topics

| Topic | Direction | Retained |
|---|---|---|
| `melb-01/ctlr/session/start` | ctlr → all | No |
| `melb-01/ctlr/session/state` | ctlr → all | Yes |
| `melb-01/ctlr/session/last` | ctlr → all | Yes |
| `melb-01/ctlr/cam/cam-01/cmd/<uuid>` | ctlr → cam | No |
| `melb-01/ctlr/cam/cam-01/status/<uuid>` | cam → ctlr | No |
| `melb-01/ctlr/phone/phone-01/sensor/<uuid>` | phone → ctlr | No |
| `melb-01/ctlr/watch/watch-01/sensor/<uuid>` | watch → ctlr | No |
| `melb-01/ctlr/can/can-01/data/<uuid>` | sensor → ctlr | No |
| `melb-01/ctlr/imu/imu-01/data/<uuid>` | sensor → ctlr | No |
| `melb-01/ctlr/gnss/gnss-01/data/<uuid>` | sensor → ctlr | No |

**QoS:** Commands `1–2` · Status/sensors `0–1` · High-frequency (CAN/IMU/GNSS) `0`

---

## 5. Storage

| Device | Format | Notes |
|---|---|---|
| `cam-01` | mp4 H.264, 30–60s snippets | rsync to Pi 4; delete on success |
| `phone-01` / `watch-01` | SQLite | Schema: `timestamp, recording_id, sensor_type, value` |
| `sensor-01` | SQLite or CSV | Same schema |
| Pi 4 aggregate | One folder per `recording_id` | Video snippets + all sensor DBs |

**Rsync**
```bash
rsync -av --remove-source-files /home/pi/trips/ pi4:/mnt/trips/
```
